# Security Policy

## Supported Versions

The following versions of AarogyaOS are currently supported with active security updates:

| Version | Supported |
| ------- | --------- |
| 1.0.x   | ✅ Yes    |
| < 1.0   | ❌ No     |

## Reporting a Vulnerability

We take the security of AarogyaOS seriously. If you find a security vulnerability, please do not report it in public GitHub issues. Instead, follow this reporting workflow:

1. Send an email to **security@aarogyaos.com** describing the issue in detail.
2. We will acknowledge receipt of your report within **48 hours**.
3. We target resolving any verified vulnerabilities within **14 days**.
4. We request that you do not publish or disclose the vulnerability until we have patched it.
5. Upon patching, we will publicly credit you in our release notes for your responsible disclosure.

## What We Commit To

*   **Prompt Responses**: We will investigate and reply to all responsible disclosures promptly.
*   **Safe Harbor**: We will not take legal action against you if you perform security testing within these guidelines and do not exploit or steal data.
*   **Encryption**: All private telemetry data and credentials are encrypted.

## Out of Scope

The following items are out of scope for our security policy:
*   Social engineering attacks against AarogyaOS team members.
*   Physical security attacks against hosting infrastructure.
*   Distributed Denial of Service (DDoS) attacks.
*   Vulnerabilities in third-party services (such as Firebase Hosting or Google Cloud Run) that are out of our direct control.
