# Google Cloud Translation API Integration

## Overview

The Smart Health (AarogyaOS) platform now includes **real-time dynamic translation** for AI-generated content using Google Cloud Translation API. This complements the existing static UI translations with live translation of agent reports, alerts, and AI responses.

## Integration Summary

### API Configuration

**Environment Variable:**
```bash
VITE_CLOUD_TRANSLATION_API_KEY=YOUR_TRANSLATION_API_KEY_HERE
```

**Service File:** `src/services/cloudTranslation.js`

### Supported Languages
- **English (en)** - Default
- **Hindi (hi)** - हिंदी
- **Tamil (ta)** - தமிழ்

## Features Implemented

### 1. Translation Service (`src/services/cloudTranslation.js`)

**Core Functions:**
- `translateText(text, targetLanguage, sourceLanguage)` - Translate single text
- `translateBatch(texts[], targetLanguage, sourceLanguage)` - Batch translation (more efficient)
- `detectLanguage(text)` - Auto-detect source language
- `translateAlert(alert, targetLanguage)` - Translate alert messages & recommendations
- `translateAgentReport(report, targetLanguage)` - Translate AI agent outputs
- `translateBotResponse(response, targetLanguage)` - Translate VaaniBot responses
- `getSupportedLanguages()` - Fetch available languages from API
- `testTranslationAPI()` - Verify API connectivity

**Key Features:**
- Automatic fallback to original text if API fails
- Checks if API key is configured before making calls
- Handles empty/null text gracefully
- Console logging for debugging

### 2. ASHA Page Integration (`src/pages/AshaPage.jsx`)

**What's Translated:**
- Gemini Vision API verification results (suspicion reasons)

**Implementation:**
- When a visit photo is verified, the AI-generated `reason` field is translated
- Translation occurs automatically when language preference is not English
- Displayed in the verification results modal

**Code Flow:**
```javascript
// After Gemini Vision verification
if (language !== 'en' && result.reason) {
  const translated = await translateText(result.reason, language, 'en');
  setTranslatedReason(translated);
}
```

### 3. Alerts Page Integration (`src/pages/AlertsPage.jsx`)

**What's Translated:**
- Alert messages (e.g., "ORS Sachets critically low at Walajah PHC")
- AI recommendations (e.g., "Transfer 500 ORS sachets from Ranipet PHC immediately")

**Implementation:**
- All active alerts are translated when language changes
- Uses batch translation for efficiency
- Maintains cache in component state to avoid re-translating
- Resolves alerts retain original language (optimization)

**Code Flow:**
```javascript
useEffect(() => {
  const translateAllAlerts = async () => {
    if (language === 'en') return;

    const translations = {};
    for (const alert of alerts) {
      const translated = await translateAlert(alert, language);
      translations[alert.id] = translated;
    }
    setTranslatedAlerts(translations);
  };
  translateAllAlerts();
}, [language, alerts]);
```

**What's Translated:**

**Implementation:**

**Code Flow:**

useEffect(() => {

## Translation Architecture

### Static vs Dynamic Translation

**Static UI Elements** (src/i18n/translations.js):
- Button labels, page titles, form fields
- Pre-translated at build time
- 367 keys × 3 languages = 1,100+ translations
- Instant switching with no API calls

**Dynamic AI Content** (src/services/cloudTranslation.js):
- Agent reports, alert messages, verification results
- Translated at runtime via Cloud Translation API
- Cannot be pre-translated (generated by AI models)

### Translation Flow

```
User Changes Language (EN → HI)
          ↓
AppContext Updates language state
          ↓
Components Detect Language Change (useEffect)
          ↓
Translate Dynamic Content via Cloud Translation API
          ↓
Update Component State with Translations
          ↓
Re-render with Translated Text
```

## Performance Optimizations

1. **Caching**: Translated content stored in component state to avoid re-translating
2. **Batch API Calls**: Multiple texts translated in single request (AlertsPage)
3. **Conditional Translation**: Only translates when language !== 'en'
4. **Graceful Fallbacks**: Returns original text if API fails
5. **Empty Text Handling**: Skips API call for empty/null text

## Error Handling

All translation functions include try-catch blocks:
- Log errors to console for debugging
- Return original text if translation fails
- Never break user experience due to translation errors

**Firebase Hosting / Cloud Run:**
```bash

```

**Docker:**
```dockerfile
ENV VITE_CLOUD_TRANSLATION_API_KEY=YOUR_TRANSLATION_API_KEY_HERE
```

### Security Considerations:

⚠️ **API Key Exposure**: The API key is visible in client-side code (Vite env variables are bundled). For production:

1. **Option 1 - API Key Restrictions (Recommended):**
   - Go to Google Cloud Console → APIs & Services → Credentials
   - Add HTTP referrer restrictions (e.g., `aarogyaos.com/*`)
   - Add API restrictions (only allow Cloud Translation API)

2. **Option 2 - Backend Proxy (Most Secure):**
   - Create Cloud Function/Cloud Run backend
   - Frontend calls backend, backend calls Translation API
   - API key never exposed to client

3. **Option 3 - Domain Restrictions:**
   - Set application restrictions to only allow your domain
   - Prevents API key theft from being useful
